# 图书商城系统  
需求规格说明书（Software Requirements Specification）

---

## 1. 引言

### 1.1 编写目的
本文档用于明确图书商城系统的功能需求和非功能需求，
作为系统设计、开发、测试及验收的依据。

### 1.2 项目范围
本系统是一个基于 Java Web 技术的图书商城系统，
实现用户浏览图书、管理购物车以及管理员后台管理功能，
不包含支付及订单处理功能。

---

## 2. 总体描述

### 2.1 用户角色
- 普通用户（USER）
- 管理员（ADMIN）

### 2.2 系统运行环境
- 操作系统：Linux
- Web 容器：Tomcat
- 数据库：MySQL
- 开发语言：Java
- 开发工具：Github Codespaces
- 版本控制：Git

---

## 3. 功能需求

### 3.1 用户功能需求

#### 3.1.1 用户注册
- 用户可以通过注册页面提交注册信息
- 注册成功后用户信息需存入数据库

#### 3.1.2 用户登录
- 用户通过用户名和密码登录系统
- 系统通过数据库校验用户信息
- 登录成功后使用 Session 维持登录状态

#### 3.1.3 用户退出
- 用户可以主动退出系统
- 系统清除对应 Session 信息

---

### 3.2 图书功能需求

#### 3.2.1 图书浏览
- 系统展示图书列表
- 支持按分类查看图书

#### 3.2.2 图书详情
- 用户可查看单本图书的详细信息

---

### 3.3 购物车功能需求

#### 3.3.1 添加购物车
- 用户可将图书加入购物车
- 同一图书重复添加时，数量增加

#### 3.3.2 查看购物车
- 用户可查看购物车中所有图书及数量

#### 3.3.3 修改购物车
- 用户可修改购物车中图书数量
- 用户可将图书从购物车中移除

---

### 3.4 后台管理功能需求

#### 3.4.1 图书管理
- 管理员可新增、修改、删除、查询图书信息

#### 3.4.2 分类管理
- 管理员可新增、修改、删除、查询图书分类

#### 3.4.3 用户管理
- 管理员可查询用户信息
- 管理员可删除用户（或禁用用户）

---

## 4. 非功能需求

### 4.1 技术约束
- 使用 Servlet、JSP、JDBC 实现系统功能
- 使用 JavaBean 封装实体对象
- 使用 Session 管理用户状态和购物车

### 4.2 数据安全
- 使用 PreparedStatement 防止 SQL 注入
- 对用户输入进行基本校验

### 4.3 可维护性
- 采用分层架构设计
- 各模块职责清晰，代码结构清楚
- Git 提交记录规范、可追溯

---

## 5. 不在范围内的功能（Out of Scope）
- 在线支付
- 订单管理
- 物流与发货
- 评论与评分
- 高并发与秒杀功能
